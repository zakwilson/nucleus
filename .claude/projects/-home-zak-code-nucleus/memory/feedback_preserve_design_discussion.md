---
name: Preserve design discussion in documents
description: When updating design docs after implementation, keep the original commentary and analysis — don't replace it with just a status note
type: feedback
---

When a feature in a design document gets implemented, add a status note but preserve the original design discussion and commentary.

**Why:** The user values the design reasoning as a record of how decisions were made, not just the outcome. The discussion remains useful context for future work even after implementation.

**How to apply:** When updating design docs post-implementation, append a **Status:** section rather than replacing the **Commentary:** section. Keep all original analysis, tradeoff discussion, and reasoning intact.
